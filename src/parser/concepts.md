| **Concept** | **Definition** | Rules |
| --- | --- | --- |
| **future_mutated_variable** | The variable will change in the program | A local variable or input parameter is written after its declaration/parameter binding. Writes are Joern assignment, compound assignment, pre/post increment, and pre/post decrement calls where the variable is the left-hand side. |
| **fixed_value** | A variable whose value remains unchanged across his full code block | A local variable or input parameter has no later write through assignment, compound assignment, pre/post increment, or pre/post decrement. Declaration initializers are not counted as later writes. |
| **gatherer** | Variable accumulates information progressively across iterations. | A local variable is declared before a `for`/`while` loop, updated inside the loop, and every detected update is an accumulating update (`x += y`, `x -= y`, `x *= y`, `x /= y`, or `x = x op y`). The non-self part of the update must contain a non-literal value. |
| **Stepper** | Steps through a systematic, predictable succession of values (e.g. a loop counter). | A local variable appears in a `for`/`while` loop condition, is updated inside that loop, and the update is predictable: pre/post increment/decrement, compound assignment with a literal, or `x = x op literal`. All detected update operators for that loop variable must be homogeneous. |
| **Walker** | Traverses a data structure element by element, where each new value depends on the previous position. | A local variable appears in a loop condition and is used inside the same loop either as an index/dereference operand (`<operator>.indirectIndexAccess`, `<operator>.indirection`) or in traversal-like calls (`next`, `hasNext`, `begin`, `end`, `find`). |
| **Most-recent holder** | Holds the latest value encountered in an unpredictable series, typically successive inputs. | A local variable is assigned inside a loop, the assignment is not a self-update, and the right-hand side contains another identifier or a stream/traversal-like producer call (`next`, `read`, `getline`, `get`, `front`, `back`, `at`). |
| **Most-wanted holder** | Holds the best value found so far (maximum, minimum, closest match). | A local variable is assigned inside an `if` nested in a loop, and the `if` condition compares the variable with another value using `<`, `<=`, `>`, or `>=`; assignments whose code contains `min`/`max` are also accepted. |
| **One-way flag** | A two-valued variable that, once changed from its initial value, never reverts (e.g. an error sentinel set to True and never reset). | A local variable is initialized with `true`, `false`, `0`, or `1`; later it is assigned the opposite literal; after that opposite assignment there is no later assignment restoring the initial literal. |
| **Organizer** | A collection used only to rearrange its elements (e.g. sorted in place), never to add or remove them. | A local variable or parameter is passed to an in-place reordering call (`sort`, `stable_sort`, `partial_sort`, `nth_element`, `reverse`, `rotate`, `shuffle`, including `std::` forms), and the same variable is not the receiver of a size-changing collection call (`push_*`, `pop_*`, `insert`, `emplace*`, `erase`, `remove`, `clear`, `append`). |
| **Container** | A collection whose elements are added and/or removed during the computation. | A local variable or parameter is the receiver/object argument of a size-changing collection call (`push_*`, `pop_*`, `insert`, `emplace*`, `erase`, `remove`, `clear`, `append`). |
| **variable_dependent** | The current identifier will be assigned with a computation that uses another variable value. | A local variable or parameter is assigned from a right-hand side whose AST contains an identifier referencing a different local variable or parameter. |
| **Follower** | Always receives the previous value of some other variable (e.g. `prev = current` before `current` is updated). | A local variable is assigned directly from another local variable or parameter, and that source variable has a later write in the same method. |
| **future_dependency** | The current value will influence future computations. | A local variable or parameter has a later read occurrence that is not the left-hand side of a write. This includes later use in expressions, calls, conditions, and returns. |
| **future_return_dependency** | The current variable will contribute to the returned value. | A local variable or parameter appears under the AST of a later `RETURN` node in the same method. |
| **variable_used_in_function** | The variable will be used as a parameter of a function | A local variable or parameter appears as argument index `>= 1` of a non-operator `CALL`. Receiver/object argument index `0` is ignored. |
| **Temporary** | Holds a value for a very short time only, most commonly in a swap operation. | A local variable participates in the canonical swap pattern `tmp = a; a = b; b = tmp` with nondecreasing source line numbers. |
| **future_branch_dependency** | The variable will later control conditional execution. | A local variable or parameter appears in the condition AST of a later `IF` or `SWITCH` control structure. |
| **single_control_flow** | The control flow block won’t have any other case | An `IF` control structure whose code does not contain an `else` branch. |
| **binary_control_flow** | The control flow block will have exactly one other case (else) | An `IF` control structure whose code contains `else` but not `else if`. |
| **n-case_control_flow** | The control flow block will have other cases (elif, switch) | A `SWITCH` control structure, or an `IF` control structure whose code contains an `else if` chain. |
| **future_loop_entry** | Current context predicts upcoming iterative execution. | A `FOR`, `WHILE`, or `DO` control structure is annotated at the loop entry line. |
| **future_recursive_call** | Current computation likely leads to recursive invocation. | A local variable or parameter is used as an argument to a call whose `NAME` or `METHOD_FULL_NAME` matches the containing method. |
| **future_branch_merge** | Distinct control-flow paths will later reconverge. | An `IF` or `SWITCH` control structure has at least two control-dependent nodes that share a later post-dominator, approximating reconvergence of branch paths in the CFG. |
